

# Система планирования проекта (соло-режим)

## Суть

После принятия заказа, перед тем как попасть в рабочий чат, игрок проходит **этап планирования** — отвечает на 5 вопросов о том, как реализовать проект. Остальные вопросы решаются автоматически с качеством 30% ("дефолт"). Ответы влияют на итоговую оценку проекта.

## Поток игры

```text
Биржа → Принять заказ → [ПЛАНИРОВАНИЕ: 5 вопросов] → Рабочий чат → Сдача
```

## Что строим

### 1. Данные вопросов (`src/lib/planningData.ts`)

Шаблоны вопросов по типам проектов. Каждый вопрос:
- `id`, `domain`, `priority`, `boss_only`, `text`
- `options[]` — варианты с `label`, `emoji`, `description`, `effects` (модификаторы quality), `industry_fit` (насколько вариант подходит индустрии заказа)

Типы проектов и количество вопросов:
- Лендинг / Портфолио / Визитка: 8-10
- Блог: 10-12  
- Интернет-магазин: 14-17
- SaaS / Корпоративный: 16-20

Стратегические (`boss_only: true`, всегда показываются):
- Подход (MVP / полная версия / быстрый хак)
- Приоритет (дизайн / функционал / перформанс)
- Коммуникация с заказчиком (показ на 50% / только при сдаче / регулярно)

### 2. Логика распределения (`src/lib/planningSystem.ts`)

Для соло-режима (без команды) алгоритм простой:
1. Взять все `boss_only` вопросы → в пул босса
2. Отсортировать остальные по `priority`
3. Добивать пул босса до 5
4. Остальное → дефолт (quality = 0.3, выбирается "безопасный" вариант)

Функция `calculatePlanningResult()` — собирает итог: что решил игрок, что ушло в дефолт, прогноз качества по аспектам (дизайн, функционал, перформанс, SEO).

### 3. Экран планирования (`src/components/PlanningPhase.tsx`)

Пошаговый интерфейс — по одному вопросу за раз (или все 5 на одном экране с прокруткой). Для каждого вопроса:
- Текст вопроса + эмодзи домена
- 2-3 варианта ответа с описанием и визуальными подсказками
- Подсветка `industry_fit` — насколько вариант подходит для индустрии заказа

После ответов — итоговый экран:
- Блок "Твои решения" (5 штук)
- Блок "Автопилот" — что ушло в дефолт с пометкой "качество 30%"
- Прогноз качества — progress-бары по аспектам
- Кнопка "Начать разработку"

### 4. Влияние на результат

`PlanningResult` сохраняется в `GameState`. При финальной оценке (`calculateReview`) учитываются:
- `industry_fit` выбранных вариантов (выбрал корпоративный стиль для пиццерии = плохо)
- Количество дефолтных решений (больше дефолтов = ниже итоговое качество)
- Модификаторы `effects` из выбранных вариантов

Формула: средневзвешенное quality всех вопросов → множитель к итоговой оценке (0.5x — 1.2x).

### 5. Интеграция в `Index.tsx`

Новое состояние между "принял заказ" и "рабочий чат":
- `gamePhase: 'board' | 'planning' | 'working'`
- После принятия заказа → показать `PlanningPhase`
- После завершения планирования → перейти в `ChatPanel`
- `planningResult` передаётся в `ReviewDialog` для расчёта

## Что НЕ делаем (пока)

- Найм сотрудников / специалисты
- Оверрайд решений специалистов
- Мораль команды
- Адаптивные стратегические вопросы для команды (time_budget, risk_management и т.д.)
