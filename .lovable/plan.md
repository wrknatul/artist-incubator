

# Система умных заказчиков — план реализации

## Что делаем

Полная переработка системы заказчиков по GDD: скрытые/видимые параметры, 7 архетипов, умный AI-промпт на основе характеристик, торг, динамическая оценка с искажениями.

## Архитектура изменений

### 1. Новая модель данных — `src/lib/clientSystem.ts`

Основной файл с типами и генератором заказчиков:

- **Типы**: `ClientArchetype`, `ClientTraits` (8 скрытых параметров: vision_clarity, tech_literacy, decision_stability, evaluation_honesty, payment_discipline, conflict_level, scope_creep, responsiveness), `ClientProfile` (видимые + скрытые + архетип + шаблоны фраз)
- **7 архетипов** с пресетами параметров: Мечтатель, Микроменеджер, Скряга, Призрак, Профессионал, Хамелеон, Токсик
- **Генератор заказов** — функция `generateOrder()`: случайный архетип → подставляются параметры с разбросом ±1-2 → генерируется бриф нужной детальности → выбираются "пропущенные детали"
- **Формула оценки** — функция `calculateReview(clientProfile, consultationCount, previewRating)`: объективная база × множитель честности ± шум от ясности видения
- **Генератор пула заказов** — `generateOrderPool(count)`: 3-5 заказов с разными архетипами

### 2. Обновление `gameData.ts`

- Расширить `FreelanceOrder` новыми полями: `deadline` (tight/flexible/yesterday), `industry`, `clientRating` (видимый, может быть фейковым), `clientProfile: ClientProfile`
- Убрать хардкод `FREELANCE_ORDERS` → заменить на динамическую генерацию через `generateOrderPool()`
- Добавить поле `negotiatedBudget` в `GameState` для торга

### 3. Обновление Edge Function `chat-client`

Ключевое изменение — system prompt строится на основе характеристик клиента:

- Передавать `clientProfile` (архетип + все скрытые параметры) в тело запроса
- System prompt динамически включает: стиль общения по архетипу, уровень детальности ответов (vision_clarity), склонность к скоупкрипу, шаблонные фразы, уровень конфликтности
- При превью: оценка искажается через evaluation_honesty (нечестный клиент занижает)
- Добавить механику scope creep: если scope_creep > 5, в промпте указание иногда просить "ещё одну мелочь"

### 4. Обновление `OrderCard.tsx`

Показывать новые видимые параметры:
- Рейтинг заказчика на бирже (⭐)
- Срок (бейдж: Жёсткий/Гибкий/Вчера)
- Отрасль
- Кнопка "Подробнее" → раскрывает бриф (детальность зависит от vision_clarity)

### 5. Обновление `FreelanceBoard.tsx`

- Генерировать пул заказов при монтировании через `generateOrderPool()`
- Хранить пул в стейте, обновлять при возврате на биржу

### 6. Обновление `ReviewDialog.tsx`

- Убрать хардкод ревью → использовать `calculateReview()` из clientSystem
- Генерировать текст отзыва по архетипу (шаблоны из GDD)
- Показывать искажённую оценку (не объективную)
- Добавить механику "защиты": кнопки торга на этапе сдачи (предложить скидку, показать метрики, защитить решения) — простые варианты с % шансом успеха

### 7. Обновление `PhoneMenu.tsx`

- Передавать `clientProfile` в чат → AI ведёт себя в соответствии с архетипом
- Добавить кнопку "Торговаться по цене" при первом открытии заказа (до начала работы) — отправляет специальное сообщение, AI реагирует по архетипу

### 8. Обновление `Index.tsx`

- `currentOrder` теперь содержит `clientProfile`
- Добавить стейт `negotiatedBudget` (результат торга)
- Передавать профиль клиента в PhoneMenu и ReviewDialog

## Что НЕ меняем

- Основной игровой цикл (биржа → работа → сдача)
- Систему генерации сайтов (ChatPanel + generate-site)
- Систему расходов и магазина
- Кат-сцену

## Детали механики торга

Минимальная реализация: при принятии заказа в чате с клиентом можно написать "хочу больше" / нажать кнопку "Торговаться". AI-клиент реагирует по архетипу (Скряга — 70% откажет, Профессионал — 50/50). Результат: `negotiatedBudget` обновляется.

## Детали механики оценки

```text
finalScore = baseScore * honestyMultiplier + noise

honestyMultiplier:
  honesty 9-10 → 1.0
  honesty 6-8  → 0.9..1.1 (random)
  honesty 3-5  → 0.7..1.0
  honesty 1-2  → 0.5..0.8

noise:
  visionClarity 8-10 → ±5%
  visionClarity 4-7  → ±10%
  visionClarity 1-3  → ±15%

baseScore = weighted sum of:
  consultations (10%) + previewShown (15%) + previewRating (75%)
```
